Entain B2B Wallet Service API (0.9)

Download OpenAPI specification:

Provides services to authenticate a player, start game sessions and perform operations on the player's wallet in response to game events. It must be implemented by the B2B Partner.

The response expects the player balance, which could be game-specific if the game_id is provided.

Authentication

Authenticate player session

path Parameters
player_id
required
string <player_id>

Player to be authenticated.

NOTE that this value cannot be trusted before authentication, as it has been passed through various layers starting from the channel client. It is up to the authentication routine to ensure that the auth token matches the player id passed here.

query Parameters
player_auth_token
required
string <player_auth_token>
Example: player_auth_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9

This token will be generated by the B2B Partner when the user logs into the client. This will be passed to this endpoint unchanged so that the game backend can verify the player and receive a new session.

game_id
string <game_id>
Example: game_id=blackjack_classic

Unique identifier of the game. This may be null or omitted if the balance request is not associated with a game, in which case the player's overall cash balance should be returned.

Responses

https://gateway.local/api//v1/wallet/player/{player_id}/authenticate

Player has performed one or more bets and/or received one or more results.

Player has performed one or more bets and/or received one or more results. Funds should be debited from the player's wallet in case of a bet, and credited to the player's wallet in case of a win. In case of jackpot wins, amounts will be provided in the jackpot_won array.

If the round_closed flag is true, the round should be marked closed.

Atomicity: If the player has multiple bets or results in the same round, they should be processed in a single transaction. Either all must fail or all must succeed. Any failed transaction should cause previous transactions to be rolled back.

Bonus

Payout a won bonus to the player

Request Body schema: application/json
required
player_id
required
string <player_id>

Unique identifier of the player.

transaction_id
required
string <transaction_id>

Unique transaction identifier for this bonus payout.

event_timestamp
required
string <date-time>

Timestamp when the bonus was bonused (ISO 8601 UTC).

bonus_type
required
string (BonusType)
Enum: "cash" "bonus_cash" "coin" "freespin" "freetoplay" "goldencoin" "freeticket"
amount
required
integer <currency> >= 0

If the the bonus type is cash or bonus_cash, this amount in 1/100 currency units (e.g., 2000 for $20.00). Otherwise, this is the raw value of the win, eg X number of free spins.

currency
string or null <currencycode>

Currency code (ISO 4217 three-letter code). NOTE This field may be null or missing if currency is not relevant to this bonus payout.

user_bonus_reference_id
required
string <transaction_id>

Unique reference ID for the bonus.

Responses
